# 光影分档画质 + 体积光 · 设计文档

- **日期**：2026-06-21
- **状态**：草案，待用户复审
- **目标**：让「光影」既**好看**又**能用**——在集成显卡上也能默认开启的一档漂亮光影，并新增**体积光(god rays)**作为核心。

---

## 1. 背景与目标

游戏已有一个布尔「光影」开关（`settings.shaders`，默认 `false`），开启时启用：真实水面（反射/菲涅尔/粼光/波纹）、太阳 PCF 投影阴影、柔和真实云、树叶随风摆。因吃集显，默认关。

用户要的是**好看且能用**：

- 把便宜、出彩的效果做成**默认就开**的一档（集显也流畅）。
- **体积光（太阳放射光束/丁达尔光）是光影最需要的**，必须做，并放进默认档。
- 水面要像我的世界光影：**偏透明的蓝 + 太阳高光/粼光 + 天空反射**（现状偏暗青绿）。
- **草木也要随风摆动**（不只树叶，还要草丛/长草）。
- **硬约束：分辨率一律不动**——游戏画面恒设备原生清晰（见 [[render-perf]] 的「分辨率最终拍板」），**体积光也不准用低分辨率内部缓冲**。性能只能靠别的杠杆。

参考：MC 光影截图（BSL / Complementary / 水反射 / 草摆动 feature）——白天水偏透明蓝+太阳粼光、太阳带 bloom/光束、草地顺风弯倒、远处起雾。

---

## 2. 范围

### 2.1 包含

1. 把单一布尔「光影」换成**三档画质**：关 / 标准 / 高（含旧存档迁移）。
2. **水面调色**：偏透明蓝 + 太阳粼光 + 天空反射 + 菲涅尔（标准/高）。
3. **草木摆动**：树叶 + 草丛/长草随风摆（标准/高）；草丛按顶点高度加权摆（根锚定、草尖摆幅大）。
4. **体积光（屏幕空间 god rays）**：太阳放射状光束，**标准 + 高都有**（高档采样更多）。
5. **太阳阴影优化**（仅高档）：收紧阴影相机 + 降 PCF 采样，让高档能用。
6. 设置 UI：「光影」勾选框 → 三选（关/标准/高）。

### 2.2 不包含（YAGNI / 留待以后）

- 真·体积光（逐像素沿光线步进采样阴影图）——太吃填充率，集显跑不动。
- SSAO、屏幕空间反射(SSR)、Bloom（除体积光本身的辉光感外）。
- 任何**降分辨率/动态分辨率**手段（硬性禁止）。
- 天气、视差背景等其他 M5 视觉项。

## 3. 画质分档模型

### 3.1 三档内容

| 档 | 水面 | 云 | 草木摆动 | 体积光 | 太阳阴影 | 面向 |
|---|---|---|---|---|---|---|
| **关** | 平贴图(经典 MC) | 方块云 | ✗ | ✗ | ✗ | 最弱机器/经典党 |
| **标准（新默认）** | 透明蓝/反射/粼光 | 柔和真实云 | ✓ | ✓(采样保守) | ✗ | 集显也流畅 |
| **高** | 同上 | 柔和 | ✓ | ✓(采样更高) | ✓(优化 PCF) | 跑得动的机器 |

判断依据：水面/云/草木摆动都便宜（水面着色只在水像素、摆是顶点级）；体积光是屏幕空间一条后处理、不依赖阴影图，可独立存在且采样可调，故进默认档；唯一最重的太阳阴影留在高档。

### 3.2 设置与迁移

- `Settings.shaders: boolean` → `Settings.lightingQuality: 'off' | 'standard' | 'high'`。
- 默认 `'standard'`。
- 迁移 `loadSettings()`：
  - 旧存档 `shaders === true` → `'high'`（他们曾主动开过全套含阴影）。
  - 旧存档 `shaders === false` 或缺失 → `'standard'`（新默认；老玩家下次进来默认就有标准光影）。
  - 非法值 → `'standard'`。
- 兼容：保留对旧字段的读取做一次性迁移；写出只写 `lightingQuality`。

> 行为变更：老玩家默认从「无光影」变为「标准光影」。这是刻意的（人人都好看），最弱机器可手动切「关」。

## 4. 各效果规格与验收

### 4.1 水面（标准/高）

改的是 `installWaterShader` 片元里的水色与反射，不动几何（水面仍平静、波纹靠程序法线流动，避免穿帮）。

- 基色从暗青绿 `vec3(0.05,0.26,0.40)` 调成**更蓝更清透**。
- 保留并调好：天空渐变反射（掠角强）、菲涅尔（俯看见底）、**流动的太阳粼光带**（随波纹碎开、`±t` 多向流动）。
- 半透明：俯看较透、掠角更不透（菲涅尔控 alpha）。

**验收**：俯看是偏透明的蓝、能隐约见底；掠角反射天空/太阳；水面有一条会动的太阳粼光；整体接近 MC 光影白天水（参考图），不再是暗青绿塑料感。

### 4.2 草木摆动（标准/高）

- 树叶：沿用现有 cutout 摆动。
- **草丛/长草**：本就渲进 cutout 批，已能蹭到摆动；改为**按顶点高度加权**——billboard 底部（贴地）位移≈0，越往草尖位移越大，像 `grass1` 参考那样顺风弯。
  - 实现：摆动位移乘以 `(顶点相对草根的高度 / 草高)`。草根=cell 底（mesher `emitPlant` 已知每片四边形底边 y）。可在 shader 里用顶点局部高度近似，或 mesher 给草顶点写一个「摆动权重」属性。
- 仅 `lightingQuality !== 'off'` 时摆（`uShaders` 门控不变）。

**验收**：开标准/高后，草丛与树叶都随风轻摆；草丛是根不动、尖摆，不是整片平移。

### 4.3 体积光 / 屏幕空间 god rays（标准 + 高）★核心

**技术**：屏幕空间放射状光散射后处理（crepuscular rays）。

1. 渲染管线改造（Renderer）：光影开时，场景先渲到一张**全分辨率** render target（颜色 + 深度），而非直接上屏。
2. 求**太阳屏幕坐标**：把太阳世界方向投影到 NDC → 屏幕 UV；太阳在视野外/地平线下则淡出/跳过。
3. **遮挡掩码复用深度**：天空（深度=远平面）为光、几何为遮挡——**不另渲一遍场景**。
4. **放射模糊**：每像素朝太阳 UV 方向取 N 个采样，按衰减累加光量 → 光束强度。
5. **叠加**：场景色 + 光束 × 太阳色（加法），输出上屏；之后再画第一人称手臂覆盖层。
6. 光影=关 时**完全跳过** RT 与后处理，直接上屏（零开销）。

**性能杠杆（全部非分辨率）**：

- **采样数 N**：标准保守（如 ~24）、高更高（如 ~48–64）；以实测帧率定。
- **太阳可见性门控**：太阳在地平线下或不在画面内 → 不跑这条后处理。
- **复用深度图**：不额外渲第二遍场景几何。

**验收**：太阳可见时，从太阳方向放射出连续的光束/丁达尔光（参考图 `day_comp`）；被地形/树挡住时光束有明暗变化；背对太阳无光束；标准档在集显上帧率可接受（实测为准）。

**风险**：全分辨率体积光在标准档对集显仍是额外一条填充率开销。缓解优先级：先减采样数 → 再加严可见性门控。若实测标准档仍扛不住，回头找用户重新权衡（不擅自降分辨率、不擅自移出标准档）。

### 4.4 太阳阴影优化（仅高）

- 收紧阴影正交相机：`SHADOW_HALF 48 → ~36`（覆盖区缩小→同 1024² 贴图下更锐、深度 pass 更省）。
- 降 PCF：片元 `3×3=9 抽样 → 4 抽样`（旋转/dither 保柔边）。
- 维持现有阴影 pass 节流（`autoUpdate=false` + `markShadowDirty`），不动。
- 具体数值（SHADOW_HALF、抽样数、是否 1024→768）实现时对着截图调。

**验收**：高档下白天地面/树有方向一致的柔和投影；集显上比现状（9 抽样、48 半宽）明显省；阴影无明显穿帮/痤疮。

## 5. 架构与改动文件

- `src/core/settings.ts`：`shaders` → `lightingQuality` 枚举 + 迁移 + 默认。
- `src/ui/settingsMenu.ts`：「光影」勾选框 → 三选控件（下拉或分段）。
- `src/main.ts`：`game.setShaders(s.shaders)` → `game.setLightingQuality(s.lightingQuality)`。
- `src/game/Game.ts`：`setShaders(on)` → `setLightingQuality(q)`，转发给 chunks/skyObjects/renderer；初值同理。
- `src/render/ChunkMeshManager.ts`：
  - `setShaders` → `setLightingQuality(q)`：`uShaders = q!=='off'`、`sun.castShadow = q==='high'`。
  - 水面片元调色（4.1）。
  - 草木摆动按高度加权（4.2，可能需 mesher 给草顶点写摆动权重属性）。
  - 阴影优化（4.4）。
- `src/render/Renderer.ts`：**新增后处理管线**（render-to-target + god-ray pass + composite + 手臂覆盖层顺序）。体积光着色器在此或独立模块。
- `src/render/SkyObjects.ts`：`setShaders` → `setLightingQuality`（云风格按 `!=='off'`）。
- `src/core/mesh/mesher.ts`（可能）：草 billboard 顶点写「摆动权重」(底 0→顶 1)，供草木摆动加权。

## 6. 性能硬约束

- **游戏画面分辨率恒 `setPixelRatio(min(devicePixelRatio,2))`，本设计一律不碰。**
- **体积光不得使用低分辨率内部缓冲**；开销只靠采样数 + 可见性门控 + 复用深度图控制。
- 光影=关 时不引入任何后处理/RT 开销（直接上屏）。

## 7. 测试策略

- **单元测试（纯逻辑）**：`settings.ts` 迁移 —— 旧 `true→high`、`false/缺失→standard`、非法→standard、默认=standard。
- **不可单测**：shader/后处理/阴影（需 WebGL）。
- **视觉 + 帧率验证**：在**独立 git worktree** 起 dev server + Playwright，逐档截图 + 测帧：
  - 水面接近 MC 透明蓝+粼光；
  - 草木摆动；
  - 体积光太阳光束出现/门控正确；
  - 标准档集显帧率可接受（重点）；高档阴影正确。

## 8. 实现分阶段（供 writing-plans 细化）

1. **设置 + 分档骨架**：settings 迁移 + UI 三选 + 全链路 `setLightingQuality` 接线（不改观感，先打通开关）。＋ settings 单测。
2. **水面调色**（4.1）。
3. **草木摆动**（4.2，含 mesher 摆动权重）。
4. **体积光后处理管线**（4.3）★最大、风险最高，独立一阶段；先打通 render-to-target + god-ray + composite + 手臂顺序，再调采样/门控/帧率。
5. **阴影优化**（4.4，高档）。
6. 逐阶段截图/测帧验证，最后整体验收 + 部署。

## 9. 并发与隔离

`ChunkMeshManager.ts`、`Game.ts`、`Renderer.ts` 是另一会话的活跃区。**本功能在独立 git worktree 实现**（见 [[using-git-worktrees]]），开发期零干扰；开发用自己的 dev server 端口 + Playwright 截图。收尾合并回 `feature/cartoon-textures` 时若与并发提交冲突再解。部署仍走既有 gh-pages 流程（见 [[github-and-deploy]]）。

## 10. 风险

| 风险 | 缓解 |
|---|---|
| 体积光全分辨率在标准档拖垮集显 | 先减采样、加严门控；实在不行回头找用户权衡（不擅自降分辨率/移出标准档） |
| 后处理管线改动影响手臂覆盖层/天空/水透明排序 | 明确渲染顺序：场景→RT、god-ray composite→屏、再画手臂；关档走原直接上屏路径 |
| 改 ChunkMeshManager/Renderer 与并发会话冲突 | worktree 隔离 + 收尾合并解冲突 + committed HEAD 自洽可构建验证 |
| 阴影优化降采样后出现穿帮/痤疮 | 调 bias + 收紧 frustum 提有效分辨率，截图验证 |
| 老玩家默认开标准光影后觉得变卡 | 提供「关」档；标准只含便宜效果 + 受控体积光 |
