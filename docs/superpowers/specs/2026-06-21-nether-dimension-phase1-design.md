# 下界维度 · 第一期「踏入下界」设计

- **日期**：2026-06-21
- **状态**：草案 v1，待用户复审
- **范围**：下界（Nether）维度的第一期——**能搭门、点燃、走进去、在有氛围的地狱里探索**。不放怪、不做流动岩浆（留第二期）。

---

## 1. 目标与非目标

### 第一期做（DoD）
1. 在主世界用**黑曜石搭 4×5 门框** + **打火石**点燃 → 紫色传送门。
2. 站进传送门几秒 → **传送到下界**（坐标 1:8）；从下界传送门可**回到主世界对应位置**。
3. 下界是一片**地狱岩 + 静态岩浆海 + 荧石 + 石英矿 + 基岩封顶封底**的大洞窟，**暗红雾、无昼夜**。
4. **黑曜石靠"水浇岩浆"获得**（忠实 MC），玩家能自己采集、搭门、循环。
5. 上/下界**各自独立存档**（方块改动、玩家位置、生物互不干扰），老存档自动迁移、不损坏。

### 第一期不做（推迟）
- 流动岩浆（第二期：扩流体系统让岩浆会流）。
- 下界怪：恶魂 / 僵尸猪人 / 岩浆怪（第二期）。
- 火焰 / 烧伤蔓延、水进下界蒸发（第二期）。
- 下界要塞、烈焰人、地狱疣、酿造（第三期）。

## 3. 架构：维度切换（最省打法）

游戏里几乎所有访问都走 `this.world.xxx`（53 处）。**不做大重构**：

- 把 `Game.world` 从 `readonly` 改为**可重指**。进/出传送门时：
  1. **保存当前维度状态**：玩家位置 → `save.playerByDimension[当前]`；生物序列化 → `save.mobsByDimension[当前]`；方块改动已按维度前缀键存（见 §5）。
  2. `this.world = new ChunkWorld(seed, 目标维度)`——`ChunkWorld` 新增 `dimension` 字段，透传给 `generateChunk` 与其生成 worker。
  3. `ChunkMeshManager.setWorld(world)`：清掉当前所有区块网格 + 换内部 world 引用（**保留 worker 池**，不重建）。
  4. 清空 `fluidSim` 活跃集；载入目标维度生物。
  5. 设玩家位置（传送门联动坐标 1:8，或该维度上次位置）。
  6. 设该维度天空/光照（§6）。
  7. 玩家周围区块重新加载（一次短暂"加载"，可接受，MC 也有传送门加载）。
- **关键点**：`fluidGrid`、`spawnWorld` 等闭包都是**每次调用读 `this.world.xxx`**（读字段、非捕获实例），重指 `this.world` 后自动跟新世界走——这正是省掉 53 处重构的原因。需核对没有闭包**捕获了旧 world 实例**（构造时核对一遍）。

> 风险：`ChunkMeshManager` 与 `Game.ts` 正被另一会话改（光影/性能）。建议下界这条线**单独/错开推进**，避免并发互相覆盖。

## 4. 世界生成：下界地形

`generateChunk(cx, cz, seed, dimension)` 加 `dimension` 分支。下界（确定性、种子加偏移避免与主世界雷同）：

- **高度**：用 y∈[0,128] 段（`CHUNK_H=192` 不变，只填下段）。
- **基岩**：y=0..2 floor、y=125..127 ceiling，**不可破坏**。
- **主体**：地狱岩（netherrack），3D 噪声**挖出大洞窟 / 连通腔**（比主世界洞穴更大更连通）。
- **岩浆海**：某"岩浆面"（如 y=31）以下、洞腔内的空气填**静态岩浆块**。
- **荧石簇**：洞顶零星生成发光荧石团。
- **石英矿**：地狱岩里嵌地狱石英矿脉。
- **灵魂沙**：零星斑块。
- 无树、无水、无草、无矿（铁煤等主世界矿不出现在下界）。

## 5. 存档：分维度

`WorldSave` 扩展（向后兼容）：

```ts
edits: Record<string, number>;            // 主世界仍用 "x,y,z"；下界用 "nether:x,y,z" → 不撞键
currentDimension?: 'overworld' | 'nether';// 当前所在维度，不存=overworld
playerByDimension?: Record<Dim, PlayerState>; // 各维度玩家位置/朝向(回程用)
mobsByDimension?: Record<Dim, SerializedMob[]>;
// furnaces 同理可加 furnacesByDimension（第一期下界无熔炉，可暂不分）
```

- **迁移**：老存档只有 `edits`（全主世界键，无前缀）→ 原样当作主世界，零改动可读。
- `editHook`/区块生成时**按当前维度过滤**应用哪批改动（前缀键天然区分）。

---

## 6. 天空 / 光照（分维度）

- `skyStateAt(time, dimension)` 加分支：下界 = **恒定暗红**（天顶暗红棕、地平线偏红橙、世界着色恒暗），**无昼夜推进**、无日月。
- `SkyObjects`（日月/云）在下界**不渲染**（或换一团静态暗红辉光）。
- 生物刷新的 `isNight` 判定：下界恒为"暗"（为第二期下界怪铺路；第一期无怪不影响）。

---

## 7. 传送门机制

- **门框**：黑曜石，MC 式 4 宽 × 5 高（内空 2×3），四角可省。
- **检测**：放黑曜石后 / 用打火石时，扫描判定是否构成合法框（纯逻辑、可单测：给定方块网格 → 是否合法框 + 内部格列表）。
- **点燃**：手持**打火石**右键框内 → 内部 2×3 填 **`nether_portal`** 块（紫色动画、非实心、可穿过、发光）。
- **传送**：玩家站在传送门块内累计 ~4 秒（或第一期简化为立即）→ 切维度。坐标：下界 = 主世界 /8；回程 ×8。**首次到达**：在对侧联动坐标找安全落点、自动生成一座黑曜石框 + 传送门。

## 8. 新内容清单

### 方块（加进 `registry.ts`）
| 方块 | 行为 |
|---|---|
| obsidian 黑曜石 | 很硬、需较高工具；门框料；水浇岩浆生成 |
| netherrack 地狱岩 | 软(0.4)，掉自身 |
| soul_sand 灵魂沙 | 减速玩家(0.5) |
| glowstone 荧石 | 自发光(光15)，掉自身 |
| nether_quartz_ore 石英矿 | 嵌地狱岩，掉石英 |
| lava 岩浆(静态) | 液体块、非实心、**碰到烧(受伤)**、橙色自发光、似水网格 |
| bedrock 基岩 | 不可破坏 |
| nether_portal 传送门块 | 非实心、发光、紫色动画、踩上去触发传送 |

### 物品
| 物品 | 行为 |
|---|---|
| flint_and_steel 打火石 | 燧石+铁锭合成；右键点燃传送门（日后可点火） |
| nether_quartz 石英 | 石英矿掉落 |

### 规则
- **水 + 岩浆 → 黑曜石**：水流入与静态岩浆块相邻/重合的格时，岩浆块转**黑曜石**、水被消耗（在水流体的扩散/方块更新路径里加一条；可单测）。
- **岩浆伤害**：玩家 AABB 与岩浆块相交 → 持续掉血（接现有受伤逻辑）。

## 10. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 并发改 `Game.ts`/渲染（另一会话光影） | 下界单独/错开推进；改动尽量集中、勤同步 |
| 重指 `this.world` 后有闭包捕获了旧实例 | 构造时逐一核对闭包都读字段；加切换冒烟测 |
| 存档迁移损坏老档 | 迁移只读旧 `edits` 当主世界，永不就地改写；保留 .bak |
| 静态岩浆=简化（不流动） | 明确是第一期取舍；第二期补流动 |
| 切维度时渲染重建卡顿 | 复用 worker 池、限量上屏（已有 flushMesh 时间预算） |
